Con esta práctica aprenderás a hacer un **Pull Request (PR)** en GitHub y a colaborar en proyectos en equipo.

## Qué es GitHub Flow

GitHub Flow es una metodología simple, flexible y eficiente para colaborar en proyectos de software. Este flujo de trabajo es ideal para equipos que desarrollan software de manera continua.

Permite a los equipos colaborar en proyectos de software de manera continua, permitiendo que los miembros del equipo trabajen en paralelo en diferentes funcionalidades o correcciones. Cuando un miembro del equipo termina su trabajo, puede hacer un **Pull Request (PR)** para solicitar que su trabajo sea fusionado con el repositorio principal.

Para facilitar la colaboración en un repositorio se suele redactar un fichero llamado **CONTRIBUTING.md** que explica las pautas a seguir para contribuir al proyecto.

## Práctica

Este repositorio incluye una web sencilla para que puedas modificarla siguiendo el flujo de trabajo de GitHub Flow.

1. Lee el fichero **CONTRIBUTING.md** para entender los pasos a seguir para contribuir a este proyecto.
2. Piensa en una funcionalidad o corrección que quieras implementar en la web. Por ejemplo, se ha dejado un enlace a una página que no existe. También puedes observar la sección de issues para darte ideas de lo que se puede mejorar.

Como este repositorio es se ha hecho para aprender a realizar una Pull Request, no se fusionarán los cambios que realices.
